## typemon

**typemon** — это проект параметрического генератора корпусов для сплит‑эргоклавиатур на Choc‑свитчах, рассчитанных под печать на фотополимерных 3D‑принтерах и генерацию модели через OpenSCAD.

Основная идея:
- **Параметрический конфиг (YAML)** описывает геометрию клавиатуры: количество колонок и рядов, углы наклона, кривизну keywell, кластеры больших пальцев, трекпойнт и т.д.
- **Шаблон OpenSCAD**, генерируемый через `go template`, превращает этот конфиг в `.scad`‑файл.
- **Go‑утилита** читает конфиг, валидирует его и подготавливает данные для генерации моделей.

Проект ориентирован на:
- сплит‑ортолинейные/колоннальные клавиатуры;
- Choc‑свитчи в хотсвап‑сокетах на маленьких одиночных PCB;
- корпуса под печать на смоляных принтерах (например, Creality Mage).

### Базовый рабочий процесс

1. Правите конфигурационный файл (например, `config.yaml` или `config.example.yaml`) под свои руки:
   - настраиваете количество колонок (от 3×5 до 4×5, с опцией дополнительного свитча на home‑ряду мизинца);
   - задаёте углы наклона основной поверхности и кластеров больших пальцев;
   - задаёте положение трекпойнта и 5‑направленной кнопки;
   - подбираете кривизну и тентинг половин.
2. Утилита на Go читает конфиг и генерирует на его основе OpenSCAD‑файл по шаблону.
3. В OpenSCAD рендерите `.scad` в STL и печатаете корпус.

### Структура репозитория (план)

- `config.example.yaml` — пример конфигурации клавиатуры.
- `go.mod`, `go.sum` — Go‑модуль для CLI‑утилиты.
- `main.go` — точка входа CLI (разбор флагов, загрузка конфига).
- `internal/config/` — структуры конфигурации и функции их загрузки из YAML.
- `templates/` — шаблоны OpenSCAD (`*.scad.tmpl`) для генерации моделей (будет добавлено позже).

### Требования

- Go 1.21+ (желательно свежая версия Go).
- OpenSCAD для визуализации и экспорта STL.
- Смоляной 3D‑принтер (например, Creality Mage) и базовый опыт работы с ним.

### Статус

Сейчас проект находится на стадии:
- определения формата конфигурации;
- реализации базового парсера YAML‑конфига в Go;
- подготовки дальнейшей генерации OpenSCAD‑шаблонов.
